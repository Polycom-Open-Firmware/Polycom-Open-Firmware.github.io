# Polycom Trio C60

The Trio C60 is the round conference phone from the same family as the
TC8: a touchscreen speakerphone with a microphone array, a proper speaker,
Wi-Fi/Bluetooth, and HDMI input. It uses the same i.MX 8M Mini SoC as the
TC8, so it runs the same Debian stack.

## Hardware

- **SoC**: NXP i.MX 8M Mini, quad Cortex-A53, 2 GiB LPDDR4
- **Display + touch**: round touchscreen
- **Audio**: microphone array and speaker
- **Radios**: Wi-Fi and Bluetooth
- **Video**: HDMI input
- **USB**: device port used for unlock and install

## Status

Hardware support covers display, touch, audio in and out, LEDs, and
Wi-Fi/BT, and the build system produces the complete C60 boot image set.
The U-Boot board port supports eMMC access and kernel handoff, and
supports dual-boot: the project's kernel and Debian in one slot, the stock
system in the other.

!!! warning "Maturity"
    The TC8 is the more mature target; C60 support is experimental.
    Releases that include C60 images list them in the release notes.

## Unlock and boot model

Unlike the TC8, the C60 is **HAB-open**: its BootROM accepts unsigned
bootloader code, so the project's U-Boot replaces the stock one directly
rather than chainloading behind it. With the chassis boot-mode switches
set to USB download, the device enumerates over USB (i.MX Serial Download
Protocol) and the [Setup Wizard](https://wizard.openpolycom.cc/) loads the
project's U-Boot from the browser via WebHID — no serial adapter at any
point. The install then overwrites the `system_a` partition (it replaces
the stock system rather than living alongside it, as the TC8 install
does). Full detail: [C60 boot chain](../internals/c60-boot.md).

## Applications

The C60 runs the same application roles as the TC8 where the hardware
allows, selected in the wizard
([choosing a role](../use/applications/index.md)).

!!! note "Smart speaker: no voice application"
    The C60-only **smart speaker** role configures the audio stack (mic
    array and speaker) but no voice application package exists: selecting
    it boots the device to a serviceable console with ssh enabled. A voice
    assistant service starts only when such a package is installed. See
    [Smart speaker](../use/applications/smart-speaker.md).
