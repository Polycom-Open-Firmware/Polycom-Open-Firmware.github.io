# Trio C60 quickstart

!!! note "Scope"
    This page covers the Trio C60. The TC8 installs differently (a
    one-time serial bootstrap) — see the [TC8 quickstart](tc8.md).

The C60 needs **no serial adapter at any point**. Its chip accepts
unsigned bootloader code, so the
[Setup Wizard](https://wizard.openpolycom.cc/) loads the project's U-Boot
into the device's RAM over plain USB (the i.MX Serial Download Protocol,
driven from the browser via WebHID), the device drops into fastboot, and
the install finishes with the same flash-and-configure flow as the TC8 —
all from one browser page.

!!! note "Release coverage"
    The wizard's C60 chooser lists the releases that ship a
    `c60-manifest.json`. The manifest's `os` section is optional, so a
    listed release does not always carry C60 OS images — if it has
    none, unlock still works and the install step reports that no C60
    OS images are in the release.

No build is required — the wizard ships the release artifacts.

## What you need

- **A Trio C60** and its power source.
- **A USB cable** from your computer to the C60's USB device port — both
  the SDP unlock and the fastboot install ride this port.
- **Chrome or Edge** on the computer (WebHID and WebUSB — Firefox and
  Safari won't work).

## Step 1 — Enter USB download mode

The C60's chassis has **BOOT_MODE switches**. Set to USB download, they
tell the chip not to boot from its internal storage at all: on power-up
it enumerates on the USB port as an i.MX recovery (SDP) device and waits
to be handed a program. The wizard's C60 flow guides this step — follow
its on-screen instructions for setting the switches and power-cycling the
device.

Nothing on the device changes in this mode; it is read-only until the
wizard explicitly flashes something. See
[what SDP loading means](index.md#what-loading-u-boot-over-usb-means-sdp).

## Step 2 — Unlock from the browser

Open the [Setup Wizard](https://wizard.openpolycom.cc/) in Chrome or Edge
and start the C60 flow. When the browser's device chooser appears, pick
the i.MX recovery device. The wizard then:

1. **Loads the project's U-Boot into RAM** over WebHID (SDP) — no `uuu`,
   no driver, no native tool.
2. **Drops the device into fastboot** — the loaded U-Boot presents a
   fastboot gadget, which the rest of the flow reaches over WebUSB.

## Step 3 — Install the OS

With the device in fastboot, pick a release. The install tail starts by
**persisting the open U-Boot** into the eMMC boot area — in the same
fastboot session — so subsequent boots run it without the USB load.
This replaces the stock bootloader directly; unlike the TC8, there is
no chainload.

!!! note "Verify the persisted bootloader"
    The flow flashes the bootloader but does not itself confirm that a
    later cold boot comes up in the open U-Boot rather than stock. After
    the install completes, confirm by power-cycling the device with the
    BOOT_MODE switches back in their normal position. If it comes up
    stock, the SDP path is always available to unlock again.

The rest of the install tail is the same as the TC8's: the wizard
flashes the slot images (`boot_a`/`dtbo_a`/`vbmeta_a`) and
sparse-flashes the root filesystem (`rootfs.simg`, an Android sparse
image) to the **`system_a`** partition. On the C60 this **replaces the
stock system** — the install overwrites `system_a` rather than living
alongside stock as the TC8 install does.

## Step 4 — Configure

The configure flow is shared with the TC8: choose the application role
the device runs at boot — including the C60-only
[smart speaker](../use/applications/smart-speaker.md) role — and set
hostname, Wi-Fi, passwords, and time zone from a form (see
[choosing a role](../use/applications/index.md) and the
[configuration reference](../use/configure.md)). The settings are applied
on the next boot.

## After install

Log in over ssh with the default credentials — and change them; see
[Getting in](../use/access.md).

To reinstall or reconfigure later, the SDP path never goes away: set the
BOOT_MODE switches to USB download again and re-run the wizard. For the
boot-model detail behind all of this, see the
[C60 boot chain](../internals/c60-boot.md).
